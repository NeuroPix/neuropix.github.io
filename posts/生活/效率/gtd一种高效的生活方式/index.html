GTD：一种高效的生活方式

本文介绍了 GTD（Getting Things Done）个人生产力系统，强调了如何有效管理任务以提升效率。在大学生活中，面对多重任务和繁杂事务，GTD 提供了一种方法，通过将任务分类到不同的分区（如 Inbox、Someday / Maybe、‘Plan Project’ Task 等），并通过五个关键流程（捕获、明晰、组织、回顾和执行）进行管理，使得个体可以将注意力集中在执行任务而非回忆任务上，从而缓解压力并提高生产力。GTD 的核心在于及时记录并组织任务，释放大脑空间，以便专注于更重要的事情。

起因

上大学之前，学习上的时间和任务都由「课程表」和「作业」来分配，我们不需要花费额外的精力来对这些任务进行管理。然而进入大学之后，随着自由可支配时间的增多，我们似乎有了更多的时间来做自己想做的事情，理应可以比高中时期更加高效地完成手头的事情。而事实却与理想大相庭径，往往一个简单的任务可能花上很长的时间才能完成，直到 Deadline 的到来；又或是在一段时间往往只能同时做一件事情，事情一多，就容易手忙脚乱，丢三落四。

例如这是我生活中的一个真实的场景：在周一，我收到老师读论文的任务并要求我在下周一进行汇报；周二在看自己博客的时候，发现有很多文章的封面没有做，得花一个时间把博客的封面制作完成；而后，因为自己刚参加完考研的初试，也需要为复试做进一步的准备，需要安排时间购买书籍，制定计划，看书等；同时又收到了辅导员开年级大会的通知；接近年末，需要花时间写一篇年度计划……这还只是其中的一部分，在生活中还有更多的其他的琐碎繁杂的事务都需要一一去处理。

不仅任务繁多，而且这些任务都各有其特点：有些任务可以在任何时候完成，而有些任务必须在某个固定的时间完成；有些任务可以一步做完，而有些任务则需要分解成多个步骤才可以做完；有些任务时机成熟，可以立即开始，而有些任务则尚需等待。

这些繁多且杂乱的任务，使得我常常花费大量的时间在「回忆」任务上，而不是「执行」任务。我也试过诸如「Apple 提醒事项」、「日历」等软件来进行提醒事项和日程管理，才发现事情远没有这么简单。直到我学习了 GTD，才暂时从这种「混乱」中逃离出来。

什么是 GTD

Getting Things Done (GTD)，或译尽管去做，是由 David Allen 开发并发表在同名书中的个人生产力系统。在该著作中，作者将该概念描述为时间管理系统，并指出若与一个任务（相关的念头停留）在一个人的脑海中，它将可能会完全填满其思绪，这将使得该人无法处理其他任务。

Getting Things Done: The Art of Stress-Free Productivity

GTD 方法的思想是将所有感兴趣的项目、相关信息、问题、任务和项目从外部记录下来，然后将它们分解为具有已知时间限制的可操作工作项目。这使人们可以将注意力集中在对外部记录中列出的每项任务采取行动上，而不是凭直觉回忆它们。

下面我将结合例子来介绍 GTD 的具体执行方法。

如何 Getting Things Done？

在 GTD 中规定了 8 个「分区」，分区是我们用来存放「任务」的地方。同时在 GTD 中包含 5 个「流程」，这 5 个流程用来对分区中的任务进行添加和管理。

GTD 分区

GTD 规定了 8 个「分区」，它们分别是：

  • Inbox
  • Someday / Maybe
  • ‘Plan Project’ Task
  • ‘Waiting For’ Task
  • ‘@Context’ Task
  • Reference
  • Trash
  • Calendar

下面分别对其进行介绍。

Inbox

「Inbox」分区相当于是一个收集箱，我们将自己在任何时候想到的任何想法，放到这个 Inbox 中。比如，看到年级群里开年级大会的通知，但暂时不对其进行安排，可以将其存入 Inbox 中。又比如我想健身，则可以在 Inbox 中创建一个名为“健身”的任务。也就是说 Inbox 可以容纳任何未经安排和处理过的任务，我们都可以先把它们扔进去，稍后再做处理。

Someday / Maybe

「Someday / Maybe」分区中通常存放一些目前不会去做但是想做的事情。比如我想未来某一天能去拉萨旅行，那么就可以在 Someday / Maybe 分区中创建这样一条任务。当然，与其称之为「任务」，不如把这个列表作为一个梦想清单。

‘Plan Project’ Task

在「’Plan Project’ Task」分区中，我们需要将一些不能一步完成的事情放在里面。比如说我计划考研，则可以在 ‘Plan Project’ Task 中写下“考研”，等到某一天在对这个 Project 做具体的规划。

‘Waiting For’ Task

在某些时候，我们的任务无法推进。比如在“制定拉萨旅行计划”的任务中，有一个问题我需要咨询导游，等待导游的回复，为了避免忘记，则需要将“制定拉萨旅行计划”这个任务移动到 ‘Waiting For’ Task 中。等到收到回复之后，再将任务从 ‘Waiting For’ Task 中移出来。

‘@Context’ Task

「’@Context’ Task」分区也称为正在执行分区，是 GTD 最核心的一个分区，我们把正在要做的、可以一步完成的任务放到 ‘@Context’ Task 中。其他分区中的任务最终都会被移动到或是被分解到 ‘@Context’ Task 终被执行。

例如在「’Plan Project’ Task」中有个任务为“做年夜饭”，我可以将其分成以下几个步骤：

  1. 询问家人想吃什么
  2. 购买食材
  3. 食材预处理
  4. 烹饪

则需要将这 4 个任务都移动到 ‘@Context’ Task 中。

总之，在 ‘@Context’ Task 的任务可以一步完成、不需要在具体的时间处理。平常时不时看看 ‘@Context’ Task，防止漏掉某些事情。

Reference

「Reference」分区用来存储一些参考资料，它是我们在完成任务的过程中所必须的资料。它通常是一些文件，在 Reference 分区中不存储任务。

Trash

并不是所有任务都一定要完成，没有完成的任务不用直接删除，将其扔到 Trash 中。日后可能的话将其从 Trash 中取出。

Calendar

在「Calendar」分区中存放需要在具体时间完成的任务。例如在周一晚上 7:00 到 8:00 召开年级大会，则直接在 Calendar 中添加一个日程即可。

上述提到的 8 个分区，是 GTD 的逻辑上的分区，可以将其存储在任何介质上。例如「Calendar」分区，这个日历可以是电子日历，当然也可以是纸质日历。

任务该放到哪个分区？

对于「Inbox」中的每个任务，我们都需要问自己 5 个问题，通过这 5 个问题的答案，将任务放到合适的分区中。

问题 1：任务是可行的吗？
如果不可行，则将其扔入垃圾箱「Trash」，或者放入「Someday / Maybe」，又或是将其归档，放入「Reference」，以后再说。

如果可行，则进入下一个问题。

问题 2：任务在一步之内可以完成吗？
如果任务不能在一步之内完成，则将其放入「’Plan Project’ Task」中，待以后对其进行分解。

如果在一步之内完成，则进入下一个问题。

问题 3：任务完成超过两分钟吗？
如果任务可以在两分钟之内完成，则立马完成它，不必添加到任务清单中。

如果不可以，则进入下一个问题。

问题 4：这件事必须由我来完成吗？
如果这件事可以由别人完成，就交给别人去做，并将其放到等待清单「’Waiting For’ Task」中。

如果必须由我来完成，则进入下一个问题。

问题 5：这件事要在特定的时间内完成吗？
如果这件事要在特定的时间内完成，则在日历「Calendar」中创建特定时间的日程。

否则，将其加入「’@Context’ Task」中，也就是执行清单。

什么时候整理任务？

在 GTD 中包含 5 个流程，它们分别是：

  • 捕获（Capture）
  • 明晰（Clarity）
  • 组织（Organize）
  • 回顾（Reflect）
  • 执行（Engage）

捕获（Capture）
在某个时间集中起来，回忆一下最近都有些什么事情需要做，以及有哪些想去做的事情。将自己的所有想法放到「

Inbox」中。

明晰（Clarity）
在 Inbox 中，把每一件事情都问自己 5 个问题，将其分到合适的分区中，明确自己要做的事情。

组织（Organize）
在此时要对分区进行管理，确保各个分区中的内容都是自己需要进行的事情。并将需要进行的事情放到相应的分区中。

回顾（Reflect）
定期回顾（可以是每日、每周或每月）各个分区的内容，确保自己不会漏掉需要完成的事情。并保持各个分区的内容都是自己需要处理的任务。

执行（Engage）
在该流程中，我们需要去执行 ‘@Context’ Task 中的任务。我们可以选择我们想做的事情来执行。

结尾

通过对 GTD 的学习，我发现 GTD 最关键的是将所有要做的事情都及时地转移到任务管理的系统中，让大脑空出更多的空间，去做那些更重要的事情。我相信这种方法可以帮助每个人更好地管理自己的生活，提升生活效率。
